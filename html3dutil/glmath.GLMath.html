(static) mat4frustum(l, r, b, t, near, far) → {Array.<number>}

Source: glmath.js, line 1533

Returns a 4x4 matrix representing a perspective projection in the form of a view frustum, or the limits in the camera's view.

This method assumes a right-handed coordinate system, such as OpenGL's. To adjust the result of this method to a left-handed system, such as in legacy Direct3D, reverse the sign of the 9th, 10th, 11th, and 12th elements of the result (zero-based indices 8, 9, 10, and 11).

Parameters:
Name Type Description
l number X-coordinate of the point where the left clipping plane meets the near clipping plane.
r number X-coordinate of the point where the right clipping plane meets the near clipping plane.
b number Y-coordinate of the point where the bottom clipping plane meets the near clipping plane.
t number Y-coordinate of the point where the top clipping plane meets the near clipping plane.
near number The distance from the camera to the near clipping plane. Objects closer than this distance won't be seen.

This value should not be 0 or less, and should be set to the highest distance from the camera that the application can afford to clip out for being too close, for example, 0.5, 1, or higher.
far number The distance from the camera to the far clipping plane. Objects beyond this distance will be too far to be seen. This value should be greater than "near" and be set so that the ratio of "far" to "near" is as small as the application can accept.

(Depth buffers often allow only 65536 possible values per pixel, which are more spread out toward the far clipping plane than toward the near plane due to the perspective projection. The greater the ratio of "far" to "near", the more the values spread out, and the more likely two objects close to the far plane will have identical depth values.)
Returns:
The resulting 4x4 matrix. (Type: Array.<number>)

(static) mat4toFrustumPlanes(matrix) → {Array.<Array.<number>>}

Source: glmath.js, line 1860

Finds the six clipping planes of a view frustum defined by a 4x4 matrix. These six planes together form the shape of a "chopped-off" pyramid (or frustum).

In this model, the eye, or camera, is placed at the top of the pyramid, four planes are placed at the pyramid's sides, one plane (the far plane) forms its base, and a final plane (the near plane) is the pyramid's chopped off top.

Parameters:
Name Type Description
matrix Array.<number> A 4x4 matrix. This will usually be a projection-view matrix (projection matrix multiplied by view matrix).
Returns:
An array of six 4-element arrays representing the six clipping planes of the view frustum. In order, they are the left, right, top, bottom, near, and far clipping planes. All six planes will be normalized. (Type: Array.<Array.<number>>)

(static) vec3cross(a, b) → {Array.<number>}

Source: glmath.js, line 79

Finds the cross product of two 3-element vectors (called A and B). The following are properties of the cross product:
  • The cross product will be a vector that is perpendicular to both A and B.
  • Switching the order of A and B results in a cross product vector with the same length but opposite direction.
  • Let there be a triangle formed by point A, point B, and the point (0,0,0) in that order. Assume the X axis points to the right and the Y axis points up. If the cross product of A and B has a positive Z component, the triangle's points are oriented counterclockwise; otherwise, clockwise. (If the X axis points right and the Y axis down, the reverse is true.) The triangle's area is half of the cross product's length.
  • If A and B are unit length vectors, (via glmath.GLMath.vec3norm), the absolute value of the sine of the shortest angle between them is equal to the length of their cross product. (More formally, the length of the cross product equals |a| * |b| * |sin θ|; where |x| is the length of vector x.)
The cross product (c) of vectors a and b is found as follows:
c.x = a.y * b.z - a.z * b.y
c.y = a.z * b.x - a.x * b.z
c.z = a.x * b.y - a.y * b.x
Parameters:
Name Type Description
a Array.<number> The first vector.
b Array.<number> The second vector.
Returns:
A 3-element vector containing the cross product. (Type: Array.<number>)
Example

The following example uses the cross product to calculate a triangle's normal vector and its area.

 var a=triangle[0];
 var b=triangle[1];
 var c=triangle[2];
 // Find vector from C to A
 var da=GLMath.vec3sub(a,c);
 // Find vector from C to B
 var db=GLMath.vec3sub(b,c);
 // The triangle's normal is the cross product of da and db
 var normal=GLMath.vec3cross(da,db);
 // Find the triangle's area
 var area=GLMath.vec3length(normal)*0.5;

(static) vec3dot(a, b) → {number}

Source: glmath.js, line 115

Finds the dot product of two 3-element vectors. It's the sum of the products of their components (for example, a's X times b's X).

The dot product has the following properties:

  • If both vectors are unit length (via glmath.GLMath.vec3norm), the cosine of the shortest angle between them is equal to their dot product. (More formally, the dot product equals |a| * |b| * cos θ where |x| is the length of vector x.) However, the resulting angle (found using the Math.acos function) will never be negative, so it can't be used to determine if one vector is "ahead of" or "behind" another vector.
  • A dot product of 0 indicates that the two vectors are orthogonal (perpendicular to each other).
  • If the two vectors are the same, the return value indicates the vector's length squared. This is illustrated in the example.
